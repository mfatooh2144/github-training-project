# Git & GitHub Project Management & Collaboration

## Project Name
MyFragmentsApp2 - Android Mobile Application

## Project Description
This project demonstrates the use of Git and GitHub for version control and collaborative development. It is based on an Android mobile application and includes practical implementation of Git workflows such as commits, branches, issues, and pull requests.

## Technologies Used
- Android (Kotlin)
- XML Layouts
- Git
- GitHub

## Project Features
- Mobile application user interface
- Fragment-based structure
- Organized project architecture
- Version control using Git

## Git Workflow Implemented
- Initialized repository using git init
- Created multiple commits (at least 5 commits)
- Created and used a feature branch (feature/design-update)
- Made UI improvements inside the branch
- Merged changes into the main branch using Pull Request

## Git Commands Used
git init
git add .
git commit -m "Initial project setup"
git branch feature/design-update
git checkout feature/design-update
git merge feature/design-update
git push origin main
git push origin feature/design-update
